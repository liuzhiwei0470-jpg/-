## 二、模块划分

### 2.1 三大模块

| 模块 | 职责 | 技术实现 |
|------|------|----------|
| **RSSHub核心** | 内容抓取、路由解析、RSS生成 | 复用RSSHub lib/routes |
| **API服务** | 用户认证、订阅管理、路由查询 | Express + TypeScript |
| **Web后台** | 前端页面、用户交互 | Vue 3 + Vite + Axios |

### 2.2 API设计

#### 认证模块 `/api/auth/*`
| 方法 | 路径 | 描述 |
|------|------|------|
| POST | /api/auth/register | 注册 |
| POST | /api/auth/login | 登录 |
| POST | /api/auth/logout | 登出 |
| GET | /api/auth/me | 获取当前用户 |

#### 订阅模块 `/api/subscriptions/*`
| 方法 | 路径 | 描述 |
|------|------|------|
| GET | /api/subscriptions | 获取订阅列表 |
| POST | /api/subscriptions | 添加订阅 |
| PUT | /api/subscriptions/:id | 更新订阅 |
| DELETE | /api/subscriptions/:id | 删除订阅 |
| GET | /api/subscriptions/:id/preview | 预览订阅内容 |

#### 路由模块 `/api/routes/*`
| 方法 | 路径 | 描述 |
|------|------|------|
| GET | /api/routes | 获取路由列表 |
| GET | /api/routes/search | 搜索路由 |
| GET | /api/routes/:name | 获取路由详情 |

#### 分类模块 `/api/categories/*`
| 方法 | 路径 | 描述 |
|------|------|------|
| GET | /api/categories | 获取分类列表 |
| POST | /api/categories | 创建分类 |
| PUT | /api/categories/:id | 更新分类 |
| DELETE | /api/categories/:id | 删除分类 |

### 3.2 数据表

#### users
| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PK, AUTO | 用户ID |

| 字段 | 类型 | 约束 | 说明 |

#### subscriptions
| 字段 | 类型 | 约束 | 说明 |

## 五、关键实现方案

### 5.1 复用RSSHub路由

```typescript
// 后端调用RSSHub核心解析
import { getRouter } from '@/lib/router';
import { parseRSS } from '@/lib/utils';

async function previewSubscription(url: string) {
  const router = await getRouter(url);
  const content = await fetchContent(url);
  return parseRSS(content);
}
```

### 5.2 用户认证

- 使用 `jsonwebtoken` 生成Token
- 密码使用 `bcrypt` 加密
- Token有效期7天
- 刷新Token机制

### 5.3 订阅预览

```
用户输入URL → 后端解析RSSHub路由 → 抓取内容 → 返回预览数据 → 前端展示
```

### 5.4 内容缓存

- 使用RSSHub现有缓存机制
- 按用户配置设置缓存时间
- 避免重复抓取

## 七、技术风险与应对

| 风险 | 应对 |
|------|------|
| RSSHub更新导致兼容问题 | 定期同步上游，接口隔离 |
| 路由解析失败 | 统一错误处理，返回友好提示 |
| 爬虫被封禁 | 限流、代理轮换、用户代理伪装 |
| 数据库性能 | SQLite适合中小规模，后期可迁移 |
