# 部署指南

## 一、环境要求

### 1.1 开发环境

| 软件 | 版本 | 说明 |
|------|------|------|
| Node.js | >= 18 | 推荐 LTS 版本 |
| pnpm | >= 8 | 包管理器 |
| SQLite | 3 | 随Node.js内置 |

### 1.2 生产环境

| 软件 | 版本 | 说明 |
|------|------|------|
| Node.js | >= 18 | LTS 版本 |
| pnpm | >= 8 | 包管理器 |
| PM2 | latest | 进程管理器 |
| Nginx | latest | 反向代理 |

---

## 二、本地开发

### 2.1 克隆项目
```bash
git clone https://github.com/xxx/rss-reader.git
cd rss-reader
```

### 2.2 安装依赖
```bash
pnpm install
```

### 2.3 配置环境变量
```bash
cp .env.example .env
# 编辑 .env 文件
```

### 2.4 启动开发服务器
```bash
# 启动后端
pnpm dev:backend

# 启动前端（另一个终端）
pnpm dev:frontend
```

### 2.5 环境变量说明

| 变量 | 说明 | 示例 |
|------|------|------|
| PORT | 服务端口 | 3000 |
| JWT_SECRET | JWT密钥 | your-secret-key |
| JWT_EXPIRES_IN | Token有效期 | 7d |
| DATABASE_URL | 数据库路径 | ./data.db |
| NODE_ENV | 环境 | development |

## 三、Docker部署

### 3.1 构建镜像
```bash
docker build -t rss-reader .
```

### 3.2 运行容器
```bash
docker run -d \
  --name rss-reader \
  -p 3000:3000 \
  -v ./data:/app/data \
  -e PORT=3000 \
  rss-reader
```

### 3.3 Docker Compose

```yaml
version: '3.8'
services:
  app:
    build: .
    ports:
      - "3000:3000"
    volumes:
      - ./data:/app/data
    environment:
      - PORT=3000
      - JWT_SECRET=your-secret-key
      - NODE_ENV=production
    restart: unless-stopped
```

启动：
```bash
docker-compose up -d
```

## 四、Nginx配置

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前端静态文件
    location / {
        root /var/www/rss-reader/frontend/dist;
        try_files $uri $uri/ /index.html;
    }

    # API代理
    location /api {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }

    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg)$ {
        root /var/www/rss-reader/frontend/dist;
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
}
```

## 五、PM2部署

### 5.1 安装PM2
```bash
npm install -g pm2
```

### 5.2 启动服务
```bash
pm2 start ecosystem.config.js
```

### 5.3 ecosystem.config.js
```javascript
module.exports = {
  apps: [{
    name: 'rss-reader',
    script: 'views/backend/dist/index.js',
    env: {
      NODE_ENV: 'production',
      PORT: 3000
    },
    instances: 1,
    autorestart: true,
    watch: false
  }]
};
```

### 5.4 PM2常用命令
```bash
pm2 list              # 查看进程
pm2 logs              # 查看日志
pm2 restart all       # 重启所有
pm2 stop all          # 停止所有
pm2 delete all        # 删除所有
```

## 八、故障排查

### 8.1 服务无法启动
```bash
# 查看端口占用
lsof -i:3000

# 查看错误日志
pm2 logs rss-reader
```

### 8.2 数据库连接失败
```bash
# 检查数据库文件权限
ls -la data.db

# 检查数据库完整性
sqlite3 data.db "PRAGMA integrity_check;"
```

### 8.3 前端无法访问API
- 检查Nginx代理配置
- 检查API服务是否启动
- 检查防火墙设置
