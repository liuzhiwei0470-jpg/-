# 接口规范

## 一、API设计原则

### 1.1 RESTful风格

| 操作 | 方法 | 路径 |
|------|------|------|
| 获取列表 | GET | /api/{资源} |
| 获取详情 | GET | /api/{资源}/:id |
| 创建 | POST | /api/{资源} |
| 更新 | PUT | /api/{资源}/:id |
| 删除 | DELETE | /api/{资源}/:id |

### 1.2 URL规范

- 使用小写字母
- 资源名词复数形式
- 路径参数用冒号 `:id`
- 查询参数用 `?key=value`

示例：
```
GET    /api/users           # 获取用户列表
GET    /api/users/:id       # 获取单个用户
POST   /api/users           # 创建用户
PUT    /api/users/:id       # 更新用户
DELETE /api/users/:id       # 删除用户
```

## 二、请求规范

### 2.1 请求头

| 头信息 | 说明 |
|--------|------|
| Content-Type | application/json |
| Authorization | Bearer {token}（需要认证时） |

### 2.2 请求参数

| 位置 | 参数格式 |
|------|----------|
| Path参数 | /api/users/:id |
| Query参数 | /api/users?page=1&limit=10 |
| Body参数 | JSON格式 |

---

## 三、响应规范

### 3.1 成功响应

```json
{
  "success": true,
  "data": { },
  "message": "操作成功"
}
```

#### 分页响应
```json
{
  "success": true,
  "data": {
    "list": [ ],
    "pagination": {
      "page": 1,
      "limit": 10,
      "total": 100,
      "totalPages": 10
    }
  }
}
```

### 3.2 错误响应

```json
{
  "success": false,
  "error": {
    "code": "ERROR_CODE",
    "message": "错误描述"
  }
}
```

### 3.3 错误码

| 错误码 | HTTP状态码 | 说明 |
|--------|-----------|------|
| VALIDATION_ERROR | 400 | 参数验证失败 |
| UNAUTHORIZED | 401 | 未登录 |
| FORBIDDEN | 403 | 无权限 |
| NOT_FOUND | 404 | 资源不存在 |
| INTERNAL_ERROR | 500 | 服务器内部错误 |

## 四、认证接口

### 4.1 用户注册
```

```

**请求参数：**
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| email | string | 是 | 邮箱 |
| password | string | 是 | 密码（6位以上） |

```json
{
  "success": true,
```

## 五、订阅接口

### 5.1 获取订阅列表
```
GET /api/subscriptions
```

**需要认证**：是

**Query参数：**
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| page | number | 否 | 页码，默认1 |
| limit | number | 否 | 每页数量，默认10 |
| categoryId | number | 否 | 分类ID |

**响应：**
```json
{
  "success": true,
  "data": {
    "list": [
      {
        "id": 1,
        "title": "B站UP主更新",
        "routeUrl": "/bilibili/user/dynamic/1",
        "categoryId": 1,
        "refreshRate": 30,
        "createdAt": "2026-06-25"
      }
    ],
    "pagination": {
      "page": 1,
      "limit": 10,
      "total": 1,
      "totalPages": 1
    }
  }
}
```

### 5.2 添加订阅
```
POST /api/subscriptions
```

**需要认证**：是

**请求参数：**
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| routeUrl | string | 是 | RSSHub路由URL |
| title | string | 是 | 订阅标题 |
| categoryId | number | 否 | 分类ID |
| refreshRate | number | 否 | 刷新频率（分钟） |
| filterKeywords | string | 否 | 过滤关键词，逗号分隔 |

### 5.3 更新订阅
```
PUT /api/subscriptions/:id
```

**需要认证**：是

### 5.4 删除订阅
```
DELETE /api/subscriptions/:id
```

**需要认证**：是

### 5.5 预览订阅内容
```

```

**需要认证**：是

**响应：**

```json
      {
        "title": "视频标题",
        "link": "https://bilibili.com/xxx",
        "content": "视频描述",
```
